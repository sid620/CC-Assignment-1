     6		documented and undocumented functions.

    12	DVINT
    13		An extract of the MSDOS Interrupt List containing the DESQview
    14		and QEMM API calls.  Updated whenever there have been significant
    15	 	additions to the DESQview/QEMM calls in the Interrupt List.
    16		Current version: DVINT43 (11/6/94)
    17	
    18	RBcomm
    19		A DESQview-aware communications program which sports seamless DSZ,
    20		PCZ, or PUMA/MPt integration for file transfers; ANSI/VT102, VT52, 
    21	 	and AVATAR emulation (including 132 columns); speeds to 115200 bps;
    22		a powerful macro language; dialing directory; ANSI music; and
    23		more.  Runs in 65K with DSZ, 46K without.  While it works under
    24		plain DOS, additional features are available under DESQview.
    25		Current version: RBCOM341 (v3.41 1/10/93)
    26	
    27	SPAWNO
    28		A replacement for the Turbo C and Microsoft C spawn..() functions
    29		which swaps the current program to disk, EMS, XMS, or raw extended
    30		memory while the spawned program executes, leaving less than 300
    31		bytes in memory.  Also includes a variant for Turbo Pascal v4.0 or
    32		higher.
    33		Current version: SPWNO413 (v4.13 12/12/92)
    34		Price: libraries free, full source code $100.
    35	
    36	RBkeyswap
    37		Exchange left control and CapsLock keys, as well as Esc and `/~.
    38		Loads itself high, using just 64 bytes; needs only 128 bytes low
    39		if no high memory is available.
    40		Current version: RBKEYSWP (v2.0 6/22/91; v3.0 included in AMISLIB)
    41	
    42	AMISLIB
    43		Public Domain library (with full source code and five sample TSRs)
    44		for creating self-highloading TSRs which may be unloaded in any
    45		order.	Up to 256 TSRs created with AMISLIB may be installed
    46		simultaneously, and TSRs can be as small as 128 bytes resident,
    47		1.4K on disk.  One of the sample TSRs is a popup which accesses
    48		DOS.
    49		Current version: AMISL091 (v0.91 10/16/92)
    50	
    51	DV-GLUE
    52		DESQview API bindings for Turbo C.
    53		Current version: DVGLUE10 (source)
    54				 DVGLUL10 (compiled)
    55				 DVGLU101 (update)   (v1.01 8/12/88)
    56		Note: a very early beta of the next release is available on
    57		FTP.CS.CMU.EDU and SoundingBoard (1/3/93)
    58	
    59	DVKPOLL
    60		"Poor man's version" of TAME.  Makes programs give up CPU time when
    61		they are in an idle loop polling the keyboard for keystrokes.
    62		Current version: DVKPOLL2 (6/18/88)
    63		Note: This program has been superseded by DVPTAME.
    64	
    65	DVPTAME
    66		"Not-so-poor man's version" of TAME.  Tells DESQview to give the
    67		remainder of a time slice away if the program is in an idle loop
    68		polling the keyboard for keystrokes.  Uses 0 bytes of memory.
    69		Current version: DVPTAME (6/4/91)
    70	
    71	DVdevload
    72		Load device drivers only in those DESQview windows which need the
    73		services of the driver, reducing the pre-DESQview overhead and thus
    74		permitting larger DESQview windows.
    75		Current version: DVDEVL12 (v1.20 1/3/93)
    76		Note: as of DESQview 2.40, Quarterdeck's DEVICE.COM is able to install
    77		  drivers into the device chain in the same manner as DVdevload;
    78		  however, DEVICE uses eight times as much memory as DVdevload and
    79		  has a serious interaction with 4DOS/NDOS that can lead to system
    80		  crashes.
    81	
    82	RBspeed
    83		Switch the CPU speed of an Intel Pentium motherboard using the
    84		"Neptune" chipset.  Public Domain.
    85		Current version: RBSPEED1 (v1.00 9/8/94)

    87	SPTURBO
    88		An integrated environment for Borland's Sprint word processor and
    89		Turbo C, Turbo Pascal, Turbo Assembler, Turbo Debugger, and CDECL.
    90		Knows which compiler to invoke based on the file's extension, and
    91		can run MAKE for multi-file projects.
    92	        Current version: SPTURBO (v2.3 Aug 1,1989)
    93		Note: Do not expect any updates, as I no longer use Sprint as a
    94		      programming environment.
