     1	Boston, MA, February 11, 1992 -- OAZ Communications today announced the
     2	availability of NetFax Director systems software, the newest member of the
     3	NetFax Open family of Fax gateways for Novell NetWare local area networks
     4	(LANs). The NetFax Open Systems are based on an "open architecture"
     5	platform, the same technology as OAZ's OEM product which is utilized by
     6	the leading long distance carriers ensuring compatibility with Fax
     7	standards worldwide.
     8	
     9	NetFax Director software runs on the NetFax Manager Fax gateway hardware
    10	and transforms an IBM PC or compatible into a Fax gateway. Up to Four
    11	NetFax boards can be configured in a NetFax Fax gateway. The NetFax Open
    12	architecture allows a multi-line server to send and receive Faxes
    13	concurrently plus provides unlimited expandability allowing multiple
    14	gateways to be added per LAN.
    15	
    16	NetFax Director can be run with NetFax DOS or NetFax Windows user
    17	interfaces or any MHS compatible E-Mail package. This "open architecture
    18	provides users with unsurpassed ease of use.
    19	
    20	The NetFax Director also provides an application program interface (API)
    21	that simplifies the integration of NetFax with vertical application
    22	programs for VARs targeting specific markets such as insurance, medical,
    23	and legal, banking/finance and manufacturing.
    24	
    25	NetFax Director delivers advanced features including: Completely automated
    26	inbound routing via DID (direct inward dialing) or semi-automatic DTMF
    27	(touch-tone decoding) routing to deliver Faxes directly to the recipient's
    28	PC; Fax Broadcasting up to two thousand using personal or global
    29	directories; Attachment of up to 64 computer files or scanned documents;
    30	Support for new fonts; HP LaserJet 2P and III printer support; HP ScanJet
    31	scanner support; 180 degree rotation for viewing faxes; Real-time
    32	conversion of outbound faxes.
    33	
    34	OAZ incorporated many powerful features into NetFax Director for network
    35	managers. OAZ developed an easy installation program plus comprehensive
    36	step-by-step documentation for administrators and users. Additionally,
    37	network managers are provided with the most comprehensive management
    38	utilities available, including: Fax queue manipulation to prioritize
    39	faxes; Fax scheduling including scheduling for off peak rates to achieve
    40	line cost savings up to sixty percent; Activity log for monitoring and
    41	reporting; departmental bill-back capabilities.
    42

    44	NetFax DOS user software for $1,995. Upgrades for O-A-Z customers to
    45	NetFax Director are only $495.

    57	Boston, MA, February 1 1, 1 992 -- OAZ Communications today announced
    58	NetFax for Windows, user software for the NetFax Open family of Fax
    59	gateways for Novell NetWare local area networks (LANs). NetFax for Windows
    60	has been designed to be fully compatible (native) with Novell's Message
    61	Handling Service (MHS) message routing platform.
    62	
    63	NetFax for Windows is the first Windows Fax program for LANs which writes
    64	Faxes directly to MHS, bypassing any gateway or font conversions.
    65	Additionally, NetFax for Windows is the first Windows-based Fax
    66	application that allows users to send multiple attachments.
    67	
    68	NetFax for Windows provides "ultra-easy" Fax capability - allowing users to
    69	Fax as easily as printing a document. Users can Fax from within any
    70	Windows PC application by simply selecting the "NetFax" option from their
    71	print menu. Plus, NetFax for Windows takes full advantage of the scalable
    72	fonts of Adobe Type Manager from Adobe Systems and Face Lift from
    73	Bitstream, providing the maximum possible resolution for Faxed messages.
    74	
    75	Additional NetFax for Windows features include: Windows 3.1 compatibility;
    76	Attachment of multiple files while print capturing from applications;
    77	Creation of broadcast group out of the active phone book; Elimination of
    78	redundant conversions via file caching during broadcasts; Import and
    79	export of telephone databases (ASCII).
